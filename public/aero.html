06 Technical FAQ

Frequently Asked Questions

Everything you need to know about the agentic dark pool protocol.

What is an agentic dark pool?

A private exchange where AI agents trade GPU compute resources without exposing order details. The protocol uses commit-reveal encryption and batch auctions to prevent MEV extraction and ensure fair, uniform pricing for all participants.

How does the batch auction work?

Orders accumulate in 30–60 second windows. A uniform clearing price algorithm matches all compatible orders simultaneously — every participant gets the same price, eliminating frontrunning, sandwich attacks, and information leakage.

What is commit-reveal and why does it matter?

Orders are encrypted client-side using Pedersen commitments before submission. No one — including the protocol itself — can see price, quantity, or direction until the batch clears. This cryptographic guarantee ensures true pre-trade privacy.

How does settlement work?

Matched orders settle atomically on Base L2 using USDC. ZK proofs verify fair execution without revealing individual order details. Typical settlement finality: under 2 seconds.

What is the x402 payment protocol?

An HTTP-native payment standard that lets AI agents pay per-request using USDC on Base. Agents include payment directly in HTTP headers — no wallets, no bridging, no human approval needed. Autonomous commerce at machine speed.

How much cheaper is this than AWS / cloud providers?

Batch auction pricing typically yields 60–80% savings vs. on-demand cloud rates by aggregating supply from independent GPU operators and eliminating intermediary margins. No reserved instances, no lock-in.

Is my order data private?

Yes. Orders are encrypted before submission using Pedersen commitments. The protocol uses zero-knowledge proofs to verify execution fairness without exposing any individual trade details. Full auditability, complete privacy.

What GPU types are supported?

H100, A100, L40S, and RTX 4090 clusters. Providers list available capacity with verifiable hardware attestations. The protocol supports both spot and reserved compute across all listed GPU classes.
